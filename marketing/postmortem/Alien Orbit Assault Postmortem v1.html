Alien Orbit Assault

How I shipped a Toaplan-style shmup using five AI tools and a lot of hand-tuning.

The Game

A browser-based vertical shmup in the Toaplan tradition. Think Truxton, Hellfire, Out Zone. You take Earth back from Gaia's corrupted orbit using three weapons and a Substrate economy. The final Orbiter boss has multiple phases, and its base only opens for 30 seconds. You have to fight to make that window appear. 5 lives. No continues.

Vertical Shmup
Toaplan-style scroll, fixed forward arc. No install. Keyboard or gamepad.
Cycler & Substrate Loop
One drifting pickup, four states. Shoot it to cycle, touch it to keep.
Built for the leaderboard
One-sitting runs. Chase the score. No saves.

AI-Powered Development Stack

CODE (light) · VS Code + Copilot Auto
Inline edits, refactors, the boring boilerplate. Auto mode picks a small model for small diffs, so I'm not paying for deep thinking on a one-line fix.
CODE (heavy) · Claude Code
My game-design partner. It built the Cycler loop, the Orbiter phase machine, and the input Binding layer, and ran most of the multi-file refactors.
ART · Nano Banana
Sprites, cover art, in-game posters. Claude wrote the prompts using the game's own words (Substrate, Orbiter, Cycler), and the style locked in quickly.
MUSIC · Suno
Three loops: title, gameplay, boss. Five passes each. I kept the take that hit hardest at low volume. Sound effects are still procedural through LittleJS's built-in ZzFX-style synth.
ENGINE · LittleJS
Tiny, fast, zero-config. The whole engine is small enough that Claude can hold all of it in context at once, which makes it surprisingly productive here.

The Art Trick That Worked

There is only one ship sprite in the build: the upright one. The tilt is faked at draw time. Split the sprite down the middle, then squish and darken the half that is diving away and stretch the other half. Flip it horizontally for the opposite direction. Free 3D-looking rotation with no extra art.

Co-developed,
not just code-generated

The best ideas came from talking it through. The Cycler replaced a four-colour loot drop because Claude pushed back on inventory bloat: "what if shooting it cycles, and touching it commits?" Done.

I asked design questions about spawn rhythm, bullet density, and how readable the safe lane was. Claude played the role of a genre expert, checking my pacing, attack cues, and reward curves.

Came together easily

  • Cycler state machine. Four states, one shootable, one consolation lock. Built right on the first try.
  • Input layer. The Action/Binding/Input source glossary went straight into the model context. Rebinding shipped clean.
  • Footer hints. One component, swaps icons based on the input source.

Still needed me: feel

Bullet spread. Ship dash i-frames. The Substrate drop curve. How fast the Orbiter's 30-second window should drain when you get hit. Screen shake on Latch Beam release.

The git history is a long chain of single-number nudges. tweak shotgun spread, then 0.18 → 0.19, then 0.19 → 0.185. Each commit looks trivial on its own. Together they are the whole game. The model can build the system, but a human still picks the numbers that feel right.

The polish tail is brutal

The last stretch was not new features. It was a long parade of small fixes that only matter once you notice them. Audio glitches, odd input states, layout overflows, and frame-perfect race conditions.

All small. None optional. Plan for it.
